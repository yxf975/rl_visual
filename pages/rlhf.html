🎯 RLHF 三阶段流水线

SFT → RM → PPO：ChatGPT / Claude / Llama3 都靠它"学会说人话"

🎯 场景介绍：一个 RLHF 三阶段全景演示。上部分三张卡展示 SFT / RM / PPO 各自在做什么，中部的流程图让你亲眼看到四个模型（π / π_ref / V / RM）之间的数据流动，下部分则通过滑动 β 看到 KL 惩罚如何在"拟合 RM 分数"和"保留原始风格"之间权衡。

阶段 1 · SFT
📚

监督微调

用高质量"指令-回答"数据微调基础模型。最大化 log P(y|x)。

输入：(prompt, 人类写的好回答)
产物：πSFT（后续所有阶段的起点）
阶段 2 · RM
⚖️

奖励模型训练

给同一 prompt 的 (chosen, rejected) 对训练一个打分器。

损失：-log σ(rw - rl)
产物：Reward Model r(x,y)
阶段 3 · PPO
🎯

策略对齐

用 RM 给回答打分，PPO 优化策略，加 KL 惩罚防偏移。

目标：max 𝔼[r(x,y)] − β·KL(π ‖ πSFT)
产物：对齐后的 π

🎬 四模型协同流水线（PPO 阶段内部）

🟡 数据流动：prompt → π 生成回答 → π_ref / RM / V 并行计算 → PPO 更新 π

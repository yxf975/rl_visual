🌳 实战选型决策树

三步问自己三个问题，一路走到叶节点，你的答案就出来了。

❓ Q1：你的任务奖励能用规则 / 测试用例自动判定吗？

比如：数学题答案对错、代码能不能过单元测试、棋类游戏输赢。

✅ Yes · 有可验证奖励
👉 首选 GRPO
省 Critic、还能省 RM，规则奖励直接当 r_i 使用，天然适合推理/代码/数学场景
❌ No · 需要人类偏好数据
❓ Q2：你是否有足够算力 + 在线反馈渠道（用户产出新数据）？

比如：你是 OpenAI/Anthropic 这种量级、或有持续的人工标注团队。

✅ Yes · 大厂 / 商业化产品
👉 选 RLHF-PPO
效果上限最高，能持续吸收新反馈，ChatGPT/Claude/Llama3 的选择
❌ No · 算力/数据有限
❓ Q3：你只拥有离线的偏好对数据吗？

比如：UltraFeedback、HH-RLHF 之类的公开偏好数据集。

✅ Yes · 离线偏好数据
👉 选 DPO
2 模型训练，像 BCE 一样稳定，开源社区最爱
🤔 No · 数据也不够
👉 先回去补 SFT + RM + 偏好数据
所有对齐算法都是在 SFT 模型和偏好数据之上建起来的大厦

🎯 典型业务场景一键对号入座

💬
通用聊天助手（2C 产品）

目标：多轮对话、礼貌、安全。需要持续迭代。

推荐：RLHF-PPO + 持续人类反馈
🧮
数学 / 代码 / 推理模型

目标：答案正确性。奖励可用判题器自动产生。

推荐：GRPO + 规则奖励（模仿 R1-Zero）
🔓
开源小模型精调

预算有限，手头有公开偏好数据集。

推荐：DPO（Zephyr 路线）
🏢
垂类企业助手（客服、法律）

专业领域，数据稀缺但严肃度高。

推荐：SFT + DPO（先稳再提）
🎨
创意写作 / 风格模型

偏好主观，难以规则化。

推荐：DPO 或 RLHF（看预算）
🛡️
安全对齐 / 红队防御

需要持续吸收新的 jailbreak 样本。

推荐：Constitutional AI / RLAIF + DPO 兜底

🚀 延伸前沿

🤖 RLAIF / Constitutional AI

用更强模型代替人类标注偏好，Claude 的核心技术。解决"人类标注贵"问题。

🎯 KTO (Kahneman-Tversky Optimization)

更进一步——只需要 thumb up/down 标签，连偏好对都不要。

🧭 ORPO · SimPO

在 DPO 基础上去掉 π_ref（只要 1 个模型）或改进长度偏置。

🔬 Process Reward Model (PRM)

对推理每一步打分（而非只给最终答案），是数学推理 + GRPO 的天然搭档。

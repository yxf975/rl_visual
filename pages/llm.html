✨ LLM 大模型中的强化学习

从 RLHF 到 DPO 到 GRPO —— 让大模型学会"说人话"的前沿对齐算法

阶段 1 · SFT
📚

监督微调

用高质量"指令-回答"数据微调基础模型。最大化 log P(y|x)。

输入：(prompt, 人类写的好回答)
产物：π_SFT
阶段 2 · RM
⚖️

奖励模型训练

给同一 prompt 的 (chosen, rejected) 对训练一个打分器。

损失：-log σ(r(chosen) - r(rejected))
产物：Reward Model r(x,y)
阶段 3 · RL
🎯

PPO 微调

用 RM 给回答打分，PPO 优化策略，加 KL 惩罚防偏移。

目标：max 𝔼[r(x,y)] - β·KL(π ‖ π_SFT)
产物：对齐后的 π

💬 交互式演示

请用一句话介绍人工智能。
回答 A（低分）

AI 就是机器人大脑，有时候会乱说话也挺搞笑的。

🎯 RM 分：2.1 📏 KL：0.8
回答 B（中分）

人工智能是让机器模拟人类思维的技术。

🎯 RM 分：5.3 📏 KL：0.3
回答 C（高分）

人工智能（AI）是研究如何让机器具备学习、推理和决策等类人智能的科学技术。

🎯 RM 分：8.7 📏 KL：0.5
📐 最终奖励：R = RM 分数 - β · KL(π ‖ π_SFT)
KL 惩罚防止策略为了刷高分而变得离谱（如重复 "好的好的好的..."）

🎁 DPO：跳过奖励模型直接优化

❌ RLHF（3 阶段，复杂）

  1. SFT：人工写回答
  2. 训练 Reward Model
  3. PPO 强化学习微调（不稳定）
😫 RM 可能有误；PPO 超参难调；训练四个模型很贵

✅ DPO（1 阶段，简洁）

  1. 直接用偏好数据优化策略！
  2. 不需要显式 RM，也不需要在线采样
  3. 本质上是个监督学习
🎉 稳定、便宜、易实现。开源社区广受欢迎
🎯 DPO 损失：
L = -log σ( β·log(π(y_w|x)/π_ref(y_w|x)) - β·log(π(y_l|x)/π_ref(y_l|x)) )

直觉：让 chosen 回答的概率比 rejected 回答更高（相对参考模型）

👥 GRPO（DeepSeek 提出）

PPO 需要额外的 Critic 网络估计 V(s)，训练大语言模型时这个代价极高。GRPO 的想法是：对同一个 prompt 生成一组 (Group) 回答，用组内奖励的归一化作为优势。

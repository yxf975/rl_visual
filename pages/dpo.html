🎯 场景介绍：一张数学推导动画带你走一遍 DPO 的漂亮闭式解： 从 RLHF 目标 → 最优策略闭式解 → 反解奖励 → 代入 Bradley-Terry → 得到 DPO loss。 再配上 logprob 演化曲线和 RLHF vs DPO 训练成本对比，一次看透 DPO 为什么这么香。

📐 从 RLHF 到 DPO：5 步闭式推导

当前步：1 / 5
1出发点：RLHF 的优化目标
maxπ 𝔼x,y∼π[ r(x,y) ] − β · KL( π(·|x) ‖ πref(·|x) )

既要最大化奖励，又要限制偏离参考模型的程度。

2最优策略的闭式解
π*(y|x) = (1/Z(x)) · πref(y|x) · exp( r(x,y) / β )

用拉格朗日乘数法可推出：最优策略是参考模型乘上指数化奖励，Z(x) 是归一化常数。

3反解奖励函数
r(x,y) = β · log( π*(y|x) / πref(y|x) ) + β · log Z(x)

把上一步两边取 log 重新整理，把奖励 r 用策略和参考模型表达出来。这一步是魔法的关键！

4代入 Bradley-Terry 偏好模型
P(yw≻yl|x) = σ( r(x,yw) − r(x,yl) ) = σ( β·log(π*(yw)/πref(yw)) − β·log(π*(yl)/πref(yl)) )

代入 r 的表达式，log Z(x) 在差值中完美抵消！这意味着我们不需要知道 Z，也不需要 RM。

5DPO Loss（最大似然即可）
LDPO = − log σ( β·log(πθ(yw)/πref(yw)) − β·log(πθ(yl)/πref(yl)) )

这就是最终的 DPO 损失函数！一个普通的监督学习目标，只需偏好对，不需要 RM，不需要采样，不需要 Critic。

💡 直觉：DPO 的"隐式奖励"

DPO 没有显式 RM，但它其实藏了一个"隐式奖励"：

r̂(x,y) = β · log( πθ(y|x) / πref(y|x) )

策略相对参考模型偏好 y 的程度，就是 DPO 心中这个 y 的奖励。

训练时 DPO 在做什么：

  • ✅ 让 r̂(yw) 上升 → π 更偏向 chosen
  • ✅ 让 r̂(yl) 下降 → π 更远离 rejected
  • 🛡️ β 控制强度：β 大 → 紧贴 π_ref；β 小 → 激进偏移

📊 训练中 logprob 如何演化

拖动 β 观察：β 越小偏移越激进，β 越大曲线越平缓。

纵轴：log π(y|x) − log π_ref(y|x)，即策略相对参考模型对 y 的偏好程度

✅ DPO 的强项

  • 🎯 训练稳定：本质是监督学习
  • 💰 成本低：只需 2 个模型，无需 Critic/RM
  • ⚡ 无需在线采样：离线偏好数据即可
  • 🛠️ 实现简单：几十行代码即可跑通
  • 📚 数据效率高：同一批偏好数据即可直接训练

⚠️ DPO 的局限

  • 🔒 依赖 π_ref 质量：SFT 不好 → DPO 上限受限
  • 📊 无法利用在线采样的新数据分布
  • 🎢 β 仍需调参，过小会导致 logprob 崩溃
  • 🔬 效果天花板略低于充分调优的 RLHF-PPO
  • 💡 对"长度偏见"敏感（可用 Length-Normalized DPO 缓解）
